My preferred Common Lisp style guide

Personally, I really like Google's Common Lisp and lisp.lang.org style guides. They are excellent compiled references if you are looking for on how to organize your code properly.

However, if you ask me, I would like some minimal changes out of both of them. Those are:

  • Append ! to a procedure name to mark it that the procedure would mutate global or variables that are passed to the procedure. Example: (defun mutator! (x) (setf x 7))
  • Use ? instead of p or -p. Example: (defun mutating? (x) (not x)) or (defun currently-mutating? (x) (not x))
Yeah. My preference looks like Scheme code style, and it's true. I really like if I can differentiate between if a function/procedure would be "destructive" in FP sense. It's also helpful to know on when to prioritize which procedure might cause trouble in unit testing as well. The biggest issue is, this style may look cluttered when you are mixing libraries code that follows the "commonly used" (similar to Google's, mostly) style.

When it comes to macros, this would be a good reference as well.

Of course, like any style guides, just don't treat them as infallible source of truth, I would say. Consistency is the key.
