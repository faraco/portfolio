Chicken Scheme as one of Golang's rivals

As someone who programs in LISP languages quite consistently, I think Chicken Scheme can do what Golang was designed for even better.

Some notable features that makes Chicken Scheme a direct contender of Golang for system services and CLI programs (IMHO):

  • Unlike Golang, Chicken Scheme is based of one of the major (and mature) LISP family languages, Scheme, a very simple and lightweight LISP.
  • Generally way more expressive with dynamic typing system, but Chicken Scheme also comes with "compile time types information analysis" if you choose to use it.
  • Static linking of eggs (modules or extensions) by default, which ease the making of a bundled executable.
  • Comes with a decent compiler which compiles Chicken Scheme source code to C.
  • Excellent C FFI, good to interact with existing C code.
  • Chicken Scheme 5.x.x core is small enough to be embeddable.
  • chicken-install to install and manage Chicken Scheme modules automatically. Think it as go get and Rust's cargo.
  • Unix-like system friendly in terms of pool of toolings and libraries.

However, Chicken Scheme 5.x.x and Scheme in general really have some significant issues as well:

  • A very tiny set of available libraries from the official egg hosting site, if not counting other available libraries by other independent authors.
  • Lack of good IDEs, if not counting Emacs which have a serious initial steep learning curve.
  • Puny community which means there are less resources and support to get from.
  • In spite of being a language with some SRFI standards, it doesn't mean you could always use other Scheme implementation code without you ending up scratching your head fixing source compatibility issues. Some Scheme languages can be too small and ended up implementing their own functions and things that might not work right away in Chicken Scheme. There is still an ongoing issue with incompatible and fragmentation of different Scheme-like languages and implementations. For example, some Guile Scheme libraries won't even work in Chibi Scheme and Chicken Scheme.
  • The mainstream programming community usually doesn't venture too far out from ALGOL syntax languages, and with uncommon LISP syntax, which could induce beginners to give them an impression of an esoteric programming language. Of course, this have nothing to do with the language itself, but this should be considered nonetheless if you are looking for contributors, for your Chicken Scheme projects.
  • Compilation time for Chicken Scheme compiler is slower compared to fast Golang's compiler. This really shows for large files.

I think Chicken Scheme would fit best for Linux kernel using distributions, FreeBSD and OpenBSD command line and background system applications at the moment, also not to forget as an embedded programming language, where currently Lua is the most popular for C/C++ interfacing lightweight embedded language. Not quite sure about execution performance between Chicken Scheme and Lua myself, however.
